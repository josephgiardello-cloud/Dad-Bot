# Failure ModelStatus: formalized failure, degradation, and divergence model for current runtime.## 1) Failure Taxonomy (What can fail)Normalized execution failures (`ExecutionFailureType`):- `LEASE_CONFLICT`- `TRACE_LOCK_CONFLICT`- `INVARIANT_VIOLATION`- `KERNEL_REJECTION`- `NODE_FAILURE`- `POLICY_REJECTION`- `UNKNOWN`Source authority: `dadbot/core/execution_failure.py`.## 2) Failure Lifecycle (How failure progresses)Every captured exception should progress through:1. `DETECTED`2. `CLASSIFIED`3. `PERSISTED`4. `RETRYABLE_DECISION`5. terminal: `TERMINAL_FAILED` or `TERMINAL_RETRIED`Source authority: `FailureStateMachine` (`dadbot/core/failure_lifecycle.py`).This gives a structured model for how failure is handled, not just where it is caught.## 3) Retry and Recovery Policy (Where it degrades vs retries)Retry eligibility is gated by:- failure retryability class- retry budget (`retry_count < max_retries`)- execution-policy profile decisions (strictness, latency pressure, conflict action)Recovery model:- durable in-flight stage marker written before node execution- deterministic stage call id injected for external idempotency checks- completed stages restored and skipped on resumeSource authority: `execution_policy.py` + `execution_recovery.py`.## 4) Degradation Model (What "degraded" means)The system degrades under policy and health pressure, not only hard failure.Primary degradation surfaces:1. policy profile selects degrade behavior instead of abort2. structural degradation markers can be emitted (`_mark_structural_degradation`)3. turn health projection captures latency/failure pressure4. kernel rejection semantics may skip/continue depending on configured actionInterpretation:- degraded execution = successful terminalization with reduced guarantees on optional/soft paths, but still with sealed trace + identity + finalization report.## 5) Divergence Model (What drift looks like)Divergence is formally detectable at three layers:1. Trace divergence- non-contiguous sequence, undeclared event type, stage-order regression, or expected-hash mismatch in execution trace contract.2. Structural divergence- missing required stages, phase-history regression, incomplete tool execution, checkpoint-chain inconsistency, ledger transaction integrity violations.3. Behavioral drift anchors- determinism manifest contract-version hash and memory-evolution fingerprints expose schema/state evolution drift across replay.If any layer fails validation, the turn is not considered fully correct.## 6) Failure Surface Map (Where to inspect first)1. `metadata.execution_failure` and `metadata.execution_failure_lifecycle`2. `state.execution_trace` and `state.execution_trace_contract`3. `state.turn_finalization_invariant_report`4. scheduler/control-plane telemetry (lease/lock conflicts, backpressure)5. recovery records (resume points, in-flight stage markers)## 7) Operational Definitions- Failure: exception path leading to `turn_failed` with terminal failure lifecycle.- Degradation: non-fatal policy/runtime pressure path that preserves sealed finalization artifacts.- Divergence: detectable mismatch in trace, structure, or deterministic replay anchors.
